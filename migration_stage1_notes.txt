SAS/C TO VBCC CONVERSION SUMMARY
---------------------------------

1. Type Changes:
   SAS/C              VBCC/AmigaOS SDK
   -----------------  -----------------
   void *             BPTR (for file handles)
   int32_t            LONG
   uint32_t           ULONG
   uint16_t           UWORD
   uint8_t            UBYTE

2. Struct Initialization:
   SAS/C:  struct Foo bar = {1, 2, 3};
   VBCC:   struct Foo bar; bar.a = 1; bar.b = 2; bar.c = 3;
   
   Rationale: VBCC in C89 mode prefers explicit initialization.
              While aggregate initialization is valid C89, VBCC's
              strict mode and the Amiga SDK headers work better
              with explicit field assignment.

3. Variable Declarations:
   SAS/C:  Allows mixed declarations and code
   VBCC:   All declarations at top of block (strict C89)
   
   Example:
   BAD:  
     int i;
     printf("test");
     int j; // Error!
   
   GOOD:
     int i;
     int j;
     printf("test");

4. Platform Guards:
   All Amiga-specific code wrapped in:
   #ifdef __AMIGA__
   ...
   #endif
   
   This allows the code to compile on both Amiga and host PC.

VBCC COMPILER FLAGS REFERENCE
------------------------------

Current flags: +aos68k -c99 -cpu=68020 -I... -D...

Flag Explanations:
+aos68k       : Target AmigaOS 68k
-c99          : Enable C99 features (while maintaining C89 base)
-cpu=68020    : Generate 68020 code (can use -cpu=68000 for compatibility)
-I<path>      : Include directory
-D<macro>     : Define preprocessor macro
-lamiga       : Link with amiga.lib (OS functions)
-lauto        : Link with auto-init code (library bases)
-lmieee       : Link with IEEE math library

Optional flags for debugging:
-g            : Generate debug info
-O0           : No optimization (for debugging)

Optional flags for warnings:
-Wall         : Enable all warnings (currently not used)
-Werror       : Treat warnings as errors
